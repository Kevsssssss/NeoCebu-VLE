System Role: Act as an Expert Full-Stack ASP.NET Core Developer and Cybersecurity Architect.

Project Context:
You are building the backend for the "Neo-Cebu 2026 Security Framework: A Context-Aware Virtual Learning Environment with QR-Based Authentication." This is a highly secure public education platform designed to mitigate "Hydra-Pulse" threats (AI-driven credential harvesting, session hijacking, deepfakes).

Tech Stack: * Backend: ASP.NET Core Web API (.NET 8/10) running natively on Windows via VS Code.

Database: Entity Framework Core (SQL Server).

Authentication: JWT (JSON Web Tokens) via ASP.NET Core Identity.

Real-time Communication: SignalR (for persistent chat).

Please adhere to the following architectural specifications and features to build this system:

IMPORTANT: STRICT RBAC & IDENTITY PROVISIONING (TEACHER ONLY)
No Student Self-Registration: There is absolutely no public registration page. Students cannot create their own accounts under any circumstances.

Teacher/Admin Provisioning: Only verified educators or administrators can generate student accounts.

Credential Generation: Upon creating a student account, the backend must generate a unique, secure payload that will be embedded into a physical QR Code (representing the student's physical school ID).

IMPORTANT: QR-BASED AUTHENTICATION FLOW
Password-less Login: The system completely eliminates student passwords. The frontend will scan the physical QR code and send the encrypted payload to the API.

Verification: The API must validate the QR payload. If valid, the system issues a secure HttpOnly JSON Web Token (JWT).

IMPORTANT: CONTEXT-AWARE ZERO TRUST PROTOCOLS
Continuous Verification: Every single API endpoint must require authorization. The JWT must be continuously verified.

Classroom Link Isolation (Context-Aware): A student's JWT is only valid for the specific virtual classroom link provisioned by their teacher. If a token is used to attempt access to an unauthorized classroom URL, the request must be instantly blocked (403 Forbidden).

Aggressive Session Timeout: The server must automatically invalidate the JWT and terminate the session after exactly 10 minutes of inactivity to prevent terminal hijacking in shared public spaces.

IMPORTANT: SECURE VIRTUAL CLASSROOM FEATURES
Encrypted Video: Setup the backend API infrastructure to securely manage tokens/rooms for a third-party WebRTC video implementation.

Persistent Chatbox: Implement SignalR WebSockets for real-time student-to-student chat. This WebSocket connection must verify the JWT on connection and strictly sanitize all inputs to prevent Cross-Site Scripting (XSS).
